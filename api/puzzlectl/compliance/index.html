Compliance evidence generation (§3.2).

Generates regulatory compliance evidence reports from audit data, agent profiles, and governance policies. Supports EU AI Act, SOC 2, ISO 27001, and NIST AI RMF frameworks.

Architecture: runs entirely in puzzlectl (client-side). puzzled stores the data; this module queries, maps, and formats evidence.

Data source: Currently reads audit records directly from the local NDJSON file (/var/lib/puzzled/audit/events.ndjson). The PRD §3.2.5 specifies D-Bus (QueryAuditEvents, ListBranches, InspectBranch) as the primary data path. D-Bus support is planned but not yet implemented — it requires async runtime integration in the compliance codepath. The local file path requires read access to the audit store (typically root). For non-root compliance queries, D-Bus support is needed (UID-scoped via QueryAuditEvents).
