puzzle-sim-worker — runs inside a sandboxed branch, executing scenario actions.

Usage: puzzle-sim-worker –scenario /path/to/scenario.yaml –root /merged

This binary is spawned by activate_branch() inside the full sandbox (namespaces, seccomp, Landlock, cgroup). It loads the scenario YAML, executes each action against the overlay-merged root, and exits. The exit code signals the outcome to the outer SimEngine.

Requires the sim feature (enabled by default). Without it, the binary prints an error and exits. For edge deployments that don’t need the simulator, compile with --no-default-features --features tui to exclude both the simulator and this binary’s implementation.
