Expand description

Manages phantom tokens — surrogate values issued to agents so that real secrets never enter the sandbox.

Fields§

§tokens: HashMap<String, PhantomToken>

Phantom tokens keyed by surrogate value.

§credential_store: Arc<RwLock<CredentialStore>>

Reference to the credential store for metadata and disk persistence.

§secure_store: Option<SecureCredentialStore>

§3.4 T3.2: mmap-backed secure store for runtime credential values. When present, real credential values are stored here (mlock’d, guard pages, MADV_DONTDUMP) rather than in the HashMap-based CredentialStore.

§phantom_prefix: String

Prefix for generated surrogate tokens.

§phantom_entropy_bytes: usize

Number of random bytes for surrogate token entropy (default: 16 per PRD §3.4).

Issue phantom tokens for a branch.

M-4: mappings is a slice of CredentialMapping structs with named fields including required (default true). When required is false and a credential is not found, it is skipped with a warning rather than failing branch creation.

Returns a list of (env_var, surrogate_value) pairs to inject into the agent’s environment.
