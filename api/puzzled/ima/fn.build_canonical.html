M21: Build canonical content for signing using JSON canonical form.

Uses BTreeMap for deterministic key ordering and serde_json::to_string for consistent serialization. Each file entry is a JSON object with sorted keys: {"checksum":"...","kind":"...","path":"...","size":N}. Files are sorted by path for deterministic ordering.

H-8: Includes agent_id, agent_profile, and checksum_before fields in the canonical output so they are covered by the Ed25519 signature.

Output format:

{"agent_id":"...","agent_profile":"...","branch_id":"...","checksum_before":"...","files":[...],"timestamp":"..."}
