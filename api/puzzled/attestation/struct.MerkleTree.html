Expand description

Append-only Merkle tree for attestation records.

Uses the frontier-based algorithm: only the right-frontier nodes (O(log n) memory) are cached for efficient appends and proof generation. The full set of leaf hashes is stored on disk for historical proof generation.

Fields§

§data_dir: PathBuf

Directory for tree data files.

§size: u64

Current tree size (number of leaves).

§frontier: Vec<Option<[u8; 32]>>

Right-frontier node hashes for efficient append. frontier[i] is the hash of the complete subtree of height i on the right edge, if one exists. None means no complete subtree at that level yet.

§leaves: Vec<[u8; 32]>

All leaf hashes in order, for proof generation. Loaded lazily from disk on first proof request.
